# JSON 格式化助手（Chrome 插件）

**一句话价值**：这是一款专为"每天都在扒日志排查线上问题"的开发者/测试/运维打造的 Chrome 扩展——你不用再小心翼翼地从时间戳、日志级别、类名一堆前缀里"精确剪出"那段 JSON，也不用把它粘到网页工具里一次次调整；只要**用鼠标随便选中一段包含 JSON 的日志**，右键点一下 **JSON 格式化**，插件就会自动从选区里**智能识别并提取出完整的 `{...}` 或 `[...]`**，在**当前页面**弹出一个可折叠、可拖动、可缩放的彩色树形窗口。它还能**同时开多个独立窗口**，让你把请求参数、上游返回、下游返回、数据库记录一屏并排对照；能自动**二次解析嵌套的转义 JSON 字符串**（就是那种 `"body":"{\"a\":1}"` 的写法）；全程**纯本地运行、不联网、不上报、无任何跨域权限**，可以放心处理线上敏感日志。一句话——把原本"复制→切标签→粘贴→格式化→切回来"的 5 步操作，压缩成"选中→右键"2 步，让你排查问题时保持在同一个上下文里，不再被 JSON 阅读打断思路。

**跟"随便一个在线 JSON 工具"比，本插件独有的 3 件事：**

- 🎯 **脏日志随便选也能解析** —— 你不用再"精确剪出" `{` 到最外层 `}`；哪怕连着时间戳、日志级别、`traceId`、类名一整行粘进来，插件也会**自动扫描出第一个完整平衡的 `{...}` 或 `[...]`**，多余前后缀统统忽略。告别"选粗了红字报错、选细了漏字段"的反复调整。
- 🪟 **多个 JSON 同屏对比排查** ——每次右键都会开出一个**独立弹窗**，可拖动、可缩放、点谁谁置顶、自动错位不重叠。请求参数 / 上游返回 / 下游返回 / DB 记录四段 JSON 可以摆成四宫格并排看，不再切标签页切到晕。
- 🪆 **嵌套转义 JSON 自动展开** —— 遇到 `{"body":"{\"orderNo\":\"X001\"}"}` 这种"字符串里套 JSON"的微服务响应，插件会**递归识别字符串值是否本身也是合法 JSON**，自动二次解析并作为子树展开，还会用小灰字标注"（嵌套 JSON）"。多层嵌套照样一层层展开，不用你手动去反斜杠。

## 为什么要做这个插件？三个真实痛点

### 痛点 1 —— "复制日志的时候要非常仔细，不然粘过去根本打不开"

日常排查线上问题，日志里的一条 JSON 通常长这样：

```
2024-08-15 10:23:41.223 INFO  [order-service,traceId=abc123] c.j.OrderCtrl - resp body = {"id":"2057293847725146112","settleNo":"...","amount":12.5}
```

想丢进在线 JSON 工具看结构？——你得**精确地**把 `{` 到最外层 `}` 之间的内容一字不差地选出来，多一个空格、少一个引号都会解析失败。选粗了？粘贴过去红字报错。选细了？漏了字段。

**本插件的做法**：随便选，把整行、甚至多行日志一起选中都行。插件会自动从选区里**扫描出第一个完整平衡的 `{...}` 或 `[...]`**，前面的时间戳、日志级别、线程名、类名统统忽略。

### 痛点 2 —— "排查问题时想同时打开好几个 JSON 对比着看"

排查一个下单失败的 bug，你手里往往有：请求参数、上游返回、下游返回、DB 里查出来的实体……四段 JSON 要同时看、来回比。开四个在线工具标签页？切来切去很快就乱。

**本插件的做法**：**每次右键都开一个独立弹窗**，每个窗口可以：

- 独立拖动到屏幕任意位置（拖标题栏即可）
- 独立缩放大小（右下角拖动）
- 自动错位排列，不会完全重叠
- 点击哪个窗口哪个就置顶

想开几个开几个，摆一屏并排对照，比切标签页顺畅得多。

### 痛点 3 —— "字符串里套着字符串，全是反斜杠转义看不清"

微服务之间传数据，经常出现"JSON 里嵌套 JSON 字符串"：

```json
{"code":0,"body":"{\"orderNo\":\"X001\",\"items\":[{\"sku\":\"A\"}]}"}
```

内层 `body` 是一个被转义过的字符串，肉眼几乎读不了结构。手动去反斜杠？——容易出错。

**本插件的做法**：识别到字符串值本身是合法 JSON 时，会**自动二次解析**并作为子树展开，同时用小灰字标注"（嵌套 JSON）"。多层嵌套照样递归下去。

## 功能特性

- ✅ **右键菜单**：选中文本后自动出现「JSON 格式化」入口
- ✅ **智能提取**：从混杂日志中提取第一个完整的 `{...}` 或 `[...]`
- ✅ **嵌套转义识别**：`"\"key\":\"value\""` 这样的转义字符串自动二次解析
- ✅ **树形视图**：折叠 / 展开 / 全部展开 / 全部折叠、显示节点数量
- ✅ **多窗口并存**：可开多个独立窗口，可拖动、可缩放、可置顶
- ✅ **原页面弹窗**：不打开新标签页，Shadow DOM 隔离样式互不干扰
- ✅ **一键复制**格式化后的 JSON
- ✅ **原文 / 树形**一键切换
- ✅ **彩色高亮**：key、字符串、数字、布尔、null 各用不同颜色
- ✅ **完全离线**：不联网、不上报、不请求任何外部权限

## 安装步骤（开发者模式加载）

1. 打开 Chrome，访问 `chrome://extensions/`
2. 打开右上角 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择本目录 `json-formatter-extension`
5. 安装完成，任意页面均可使用

> 需要更新时，回到扩展页点击本扩展右下角的刷新图标即可。

---

## 使用方法

1. 在网页上（比如你的日志页面）用鼠标选中一段包含 JSON 的文本，比如：

   ```
   2024-08-15 10:23:41 INFO resp = {"id":"2057...","settleNo":"...","amount":12.5}
   ```

2. 右键 → 点击 **JSON 格式化**
3. 当前页面立即弹出格式化窗口
4. 顶部按钮：
   - **全部展开 / 全部折叠**
   - **复制** 格式化后的 JSON
   - **查看原文 / 查看树形** 之间切换
   - **关闭**

### 多窗口对比技巧

- **同一页面反复右键**即可叠开多个窗口，每个窗口互相独立
- 按住标题栏可拖动，右下角小三角可缩放
- 点击任意窗口，该窗口会**自动置顶**到最上层
- 排查时建议摆成"上下左右"四宫格：请求参数 / 上游响应 / 下游响应 / DB 记录

## 隐私与权限

本扩展**完全在本地运行**，你可以放心处理敏感日志：

| 请求的权限 | 用途 | 说明 |
|---|---|---|
| `contextMenus` | 注册右键菜单 | 仅在选中文本时才显示菜单 |
| `scripting` | 向当前页注入弹窗代码 | 只在点击菜单时执行 |
| `activeTab` | 访问当前活动标签页 | 仅当前 tab，不后台监听 |

**没有请求**：`host_permissions`、`storage`、任何网络权限、任何跨域访问。
**不会发生**：不采集、不上传、不同步、不联网、不记录你的选中内容。

## 常见问题（FAQ）

**Q：为什么点了右键菜单没反应？**
A：多为宿主页面 CSP 拦截脚本注入。请留意浏览器控制台是否有 `[JSON 格式化助手] 注入失败` 报错。极少数强 CSP 站点（如 GitHub 的部分页面）会阻止扩展注入。

**Q：能格式化整个页面的 JSON 响应吗（像浏览 `.json` URL 那样）？**
A：本插件专注**选中文本 → 弹窗展示**这个场景，不接管整个页面渲染。如需自动格式化 `.json` 响应，请配合官方 JSON Viewer 类扩展使用。

**Q：可以处理多大的 JSON？**
A：正常业务日志（几十 KB 到几 MB）无压力。超大 JSON（几十 MB）建议不要选中一大段，浏览器 selection 本身就有卡顿风险。

**Q：为什么弹窗样式没有被网页 CSS 影响？**
A：弹窗使用 Shadow DOM 隔离，宿主页面的 CSS 无法穿透进来，弹窗内部样式也不会污染宿主页面。
